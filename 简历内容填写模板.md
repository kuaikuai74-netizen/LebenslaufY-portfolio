# 简历内容填写模板

这份模板的目标不是把信息填满，而是帮你把“招聘方最关心的内容”写清楚。

很多 GitHub 个人主页和求职作品集的共同点是：

- 开头先说明你是谁、想做什么、擅长解决什么问题
- 经历部分强调结果，而不是只写职责
- 项目部分用具体案例证明能力
- 技能部分只写和目标岗位强相关的内容

## 1. 首页一句话介绍

套用模板：

`我是 [身份/岗位]，擅长解决 [问题类型]，希望在 [目标方向] 中持续创造 [结果/价值]。`

示例：

`我是有 3 年经验的产品运营，擅长把复杂项目拆解成可执行流程，希望在内容增长与用户体验优化方向持续创造可量化结果。`

## 2. 个人摘要

建议控制在 2 到 4 句。

可直接填写：

- 我目前的身份：
- 我最擅长的 2 到 3 件事：
- 我做过的代表性成果：
- 我现在希望寻找的机会：

示例结构：

`我有 [X] 年 [行业/岗位] 经验，长期负责 [职责范围]。`
`我擅长 [能力1]、[能力2]、[能力3]。`
`过去我曾推动 [项目/业务] 实现 [量化结果]。`
`我希望寻找 [岗位方向/团队类型] 的机会。`

## 3. 经历填写法

每段经历尽量包含下面四项：

- 时间：
- 公司 / 团队：
- 岗位：
- 1 到 3 条结果导向描述：

推荐句式：

`负责 [事项]，通过 [动作]，达成 [结果]。`

`在 [背景/挑战] 下，主导 [动作]，最终实现 [量化结果]。`

`协同 [团队/角色] 完成 [项目]，将 [指标] 从 [原值] 提升到 [新值]。`

避免这样写：

`负责日常运营工作。`
`参与多个项目推进。`
`完成领导交办的任务。`

因为这些句子说明了“做过”，但没有说明“做成了什么”。

## 4. 项目填写法

每个项目建议回答这 4 个问题：

- 项目背景是什么？
- 你在里面扮演什么角色？
- 难点是什么？
- 最后的结果是什么？

可直接填写：

### 项目名称

- 背景：
- 角色：
- 动作：
- 结果：
- 可量化数据：
- 链接（如果有）：

推荐压缩成网页上的 2 到 3 句：

`针对 [问题]，我负责 [角色/范围]，通过 [关键动作]，最终实现 [结果]。`

## 5. 技能填写法

建议分成 3 组，而不是一口气列很多名词：

- 专业能力：例如产品设计、数据分析、内容策划、项目管理
- 工具与技术：例如 Figma、Excel、SQL、React、Python
- 工作方式：例如结构化表达、独立推进、跨团队协作

判断要不要写某个技能，可以问自己：

`如果面试官追问这个技能，我能讲出案例吗？`

如果不能，先别写。

## 6. 联系方式

建议保留这些信息中的 2 到 4 项：

- 邮箱
- 手机号
- 微信
- GitHub
- LinkedIn
- 作品集链接
- PDF 简历链接

## 7. 最终检查清单

发布前可以快速检查：

- 首页 10 秒内能看懂你是谁
- 经历里至少有几个量化结果
- 项目描述里能看出你的具体角色
- 技能没有堆砌过多术语
- 联系方式清晰可用
- 手机端浏览没有排版问题
